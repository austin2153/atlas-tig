# Fresh Installation Guide

## Zero-Configuration Deployment

This stack is designed to deploy with **no manual configuration required**. All credentials are auto-generated securely.

### Quick Start

1. **Clone the repository**
   ```bash
   git clone <your-repo-url>
   cd atlas-tig
   ```

2. **Run the deployment script**
   ```bash
   ./deploy.sh
   ```

That's it! The deployment script will:
- Copy `.env.example` to `.env`
- Auto-generate secure Grafana admin password (saved to `.env`)
- Auto-generate InfluxDB admin token (saved to `.env`)
- Create all configuration files from templates
- Start all services

### What Gets Created Automatically

On first run, `deploy.sh` creates:
- `.env` file with auto-generated credentials:
  - `GRAFANA_ADMIN_PASSWORD` (32-character random password)
  - `INFLUXDB_TOKEN` (InfluxDB API token)
- `influxdb-admin-token.txt` (gitignored)
- `influxdb-explorer/config/config.json` (gitignored)

### Access the Stack

After deployment completes:
- **InfluxDB Explorer**: http://localhost:8888 (auto-connected, no login needed)
- **Grafana**: http://localhost:3000
  - Username: `admin`
  - Password: Check `.env` file for `GRAFANA_ADMIN_PASSWORD`

**To view your Grafana password:**
```bash
grep GRAFANA_ADMIN_PASSWORD .env
```

### Credentials Management

All credentials are stored in `.env` (gitignored for security):
```bash
# Example .env content after first deployment:
GRAFANA_ADMIN_PASSWORD=<auto-generated-32-char-password>
INFLUXDB_TOKEN=apiv3_<auto-generated-token>
```

**Important:** Keep your `.env` file secure! These are production credentials.

### Optional Configuration

#### Custom Passwords (Before First Run)

If you want to set custom passwords instead of auto-generated ones, create `.env` before running `./deploy.sh`:

```bash
# Create .env with custom passwords
cat > .env << 'EOF'
GRAFANA_ADMIN_PASSWORD=your-custom-password-here
EOF

# Then run deployment (will skip password generation since .env exists)
./deploy.sh
```

The InfluxDB token will still be auto-generated, but Grafana will use your custom password.

### Complete Wipe & Redeploy

To test a completely fresh installation:
```bash
# Stop and remove everything
docker compose down -v

# Remove generated files to commit):
- `.env.example` - Template with commented placeholders
- `influxdb-explorer/config/config.json.template` - Token placeholder
- `compose.yml`, `deploy.sh`, configs

**Generated files** (gitignored, never committed):
- `.env` - Contains auto-generated credentials
- `influxdb-admin-token.txt` - Generated from .env
- `influxdb-explorer/config/config.json` - Generated from template

### Security Best Practices

✅ **DO:**
- Keep `.env` file secure with proper file permissions
- Back up `.env` to secure password manager
- Rotate passwords periodically

❌ **DON'T:**
- Commit `.env` to git (already in `.gitignore`)
- Share credentials in plain text
- Use default `admin/admin` in production (we auto-generate secure passwords!)

### Troubleshooting

**Forgot Grafana password?**
```bash
# View password from .env
grep GRAFANA_ADMIN_PASSWORD .env
```

**Need to reset all credentials?**
```bash
# Stop and wipe everything
docker compose down -v
rm .env influxdb-admin-token.txt influxdb-explorer/config/config.json

# Redeploy with new auto-generated credentials
./deploy.sh
```
**Template files** (safe):
- `.env.example` - Documentation only
- `influxdb-explorer/config/config.json.template` - Token placeholder
- `compose.yml`, `deploy.sh`, configs

**Generated files** (gitignored):
- `.env` - Contains secrets
- `influxdb-admin-token.txt` - Generated from .env
- `influxdb-explorer/config/config.json` - Generated from template
