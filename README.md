# InfluxDB 3 Core Deployment

This project uses InfluxDB 3 Core with automated token management.

## Quick Start

### First Time Deployment

Just run:
```bash
./deploy.sh
```

The script will:
1. Detect that no token exists
2. Start InfluxDB
3. Generate a new admin token automatically
4. Save it to `.env`
5. Start all services

### Subsequent Deployments

Same command:
```bash
./deploy.sh
```

The script will use the existing token from `.env`.

### Complete Fresh Start

To completely redeploy (wipe all data):

```bash
docker compose down -v  # Remove containers and volumes
./deploy.sh             # Deploy fresh (will reuse token from .env)
```

## Configuration

Edit `.env` to add your UniFi credentials:
```bash
INFLUXDB_TOKEN=<auto-generated>
UNIFI_API_KEY=your_unifi_key
UNIFI_CONSOLE_ID=your_console_id
```

The `INFLUXDB_TOKEN` is automatically created on first deployment.

## Access Points

- **InfluxDB**: http://rocky-linux.atlas.local:8181
- **Grafana**: http://rocky-linux.atlas.local:3000 (admin/admin by default)

## Stack Components

- **InfluxDB 3.9.0-core**: Time-series database
- **Telegraf 1.38.2**: Metrics collection
- **Grafana 11.0.0**: Visualization

## Important Files

- `.env`: Contains all secrets (NEVER commit this!)
- `influxdb-admin-token.txt`: Auto-generated from `.env` (already in .gitignore)
- `deploy.sh`: Smart deployment script that handles first-time setup
